  1. 1

    Brand header

    Gradient bar (#2a1d4f → #874ffa) with logo + tagline. Same height across every email so the brand silhouette is recognisable in the inbox preview.

  2. 2

    Hero

    Coloured badge (.badge.ok / .warn / .danger) + h1 + 1-line summary. The hero alone should answer "what is this email about?"

  3. 3

    Body copy

    Conversational paragraphs with merge fields like {{firstName}}. Use sparingly — most transactional emails should fit in two short paragraphs.

  4. 4

    Stats / property / list

    Optional structured block: .em-stats for numerics, .em-prop for a listing card, .em-list for line items. Pick at most one.

  5. 5

    Primary CTA

    Single brand-coloured button. The fallback URL underneath is required for accessibility and for clients that strip buttons.

  6. 6

    Notice (optional)

    Inline callout for warnings, expirations, or security details. Three variants: amber (caution), red (danger), blue (info).

  7. 7

    Signature

    For human-sent emails (loan officer, agent). Skip on system emails.

  8. 8

    Compliance footer

    Dark footer with brand line, legal links, mailing address, copyright + NMLS. Required by CAN-SPAM and lender regulations.

2

Block library

Modular pieces — mix and match per email type.

Header

.em-head

Gradient brand strip. Always present. Same on every template.

Used in: all 38 templates

Hero

.em-hero

Badge + headline + summary. The "TL;DR" of the email.

Used in: all 38 templates

Body copy

.em-body

Free-form paragraphs with merge tags and links.

Used in: 36 / 38

Stats strip

.em-stats

3-column key/value/detail. For amounts, dates, IDs.

Used in: receipts, milestones, statements

Property card

.em-prop

Photo + ref + name + address + price.

Used in: listing alerts, deal milestones

Line item list

.em-list

Key/value rows with optional total. Bordered table.

Used in: statements, receipts, schedules

Primary CTA

.em-cta

Single brand button + fallback URL. Always one per email.

Used in: 35 / 38

Notice

.em-notice

Amber, red, or blue inline callout for warnings or expirations.

Used in: security, expirations, action-required

Signature

.em-sig

Name + role for emails sent on behalf of a person.

Used in: human-sent emails (agents, officers)
